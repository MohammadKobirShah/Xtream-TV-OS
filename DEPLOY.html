Pro-Tip #2 · Admin Controlled

⚡ FFmpeg Restream — 3 Modes

Controlled from Settings → FFmpeg Global Mode. Default is OFF for maximum performance. Override per-channel or globally.

🟢
OFF — Default
fpassthru() pipe. Zero CPU overhead. Kernel-level sendfile. Perfect for 4K. Best for 99% of streams.
🔴
ON — FFmpeg Always
Every stream goes through FFmpeg. Fixes broken timestamps, re-muxes to TS, bypasses geo/CDN blocks.
🟡
AUTO — Smart Fallback
Probes upstream first. Uses passthru if reachable. Falls back to FFmpeg automatically if stream fails.
⚙️ Control Priority Chain
Priority (highest → lowest)
# 1. URL override — for testing only, no DB write
proxy.php?url=BASE64&t=TOKEN&mode=off|on|auto

# 2. Per-channel override — set from Settings → Channel Table
channels.ffmpeg_mode = inherit|off|on|auto

# 3. Global setting — set from Settings panel
settings.ffmpeg_mode = off  (default)
Quality Presets
passthru → copy codecs, remux only (fastest, 4K safe)
hd       → scale 1280×720, H264 CRF23 + AAC 128k
sd       → scale 854×480, H264 CRF28 + AAC 96k
FFmpeg Flags (auto-applied)
-reconnect 1           stream auto-reconnect
-reconnect_streamed 1  reconnect on error
-user_agent SmartTV    geo-block bypass UA

Docker · Day-to-Day

🐳 Management Commands

Every command you'll ever need to operate your IPTV OS.

▶ Start / Build
docker compose up -d --build       # rebuild image + start (after code change)
docker compose up -d               # start existing containers
docker compose restart iptv-os     # restart PHP app only
docker compose restart cloudflared # restart tunnel only
⏹ Stop / Clean
docker compose down               # stop all (data preserved in volumes)
docker compose down -v             # ⚠️ stop + DELETE volumes (wipes DB!)
docker compose down --rmi all      # stop + delete built images
🔍 Debug / Monitor
docker compose logs -f            # live logs all containers
docker compose logs -f cloudflared # cloudflare tunnel status
docker compose logs -f iptv-os    # PHP app logs
docker exec -it kobir_iptv_os bash # shell into PHP container
docker stats                       # live CPU / RAM usage
docker compose ps                  # status + health check state
💾 Backup & Restore
# Backup SQLite database
docker cp kobir_iptv_os:/var/www/html/xtreamtv/storage/database.sqlite \
  ./backup_$(date +%Y%m%d_%H%M).sqlite

# Restore from backup
docker cp backup_20250527.sqlite \
  kobir_iptv_os:/var/www/html/xtreamtv/storage/database.sqlite

# Dump as SQL
docker compose exec iptv-os sqlite3 \
  /var/www/html/xtreamtv/storage/database.sqlite .dump > backup.sql
🔄 Rotate Cloudflare Token
# 1. Get new token from Cloudflare Zero Trust dashboard
# 2. Update .env file:
CLOUDFLARE_TUNNEL_TOKEN=new_token_here

# 3. Restart only the tunnel container (no rebuild needed)
docker compose restart cloudflared

Before You Go Live

🛡️ Security Checklist

⚠️
Production Security — Do These Before Exposing to the Internet
1. Change default admin password (admin123) immediately via Users panel.
2. Delete or rename install.php after first setup.
3. Keep .env in .gitignore — never commit your Cloudflare token.
4. Enable firewall — only expose port 8080 to trusted IPs if not using Cloudflare Tunnel.
5. Rotate your API tokens periodically from the Users panel.
6. Check FFmpeg mode — leave OFF unless you need geo-block bypass.
💡
Using Cloudflare Tunnel? You don't need to open any ports. Cloudflare handles SSL/TLS termination and DDoS protection automatically. Your IPTV OS is accessible via a *.trycloudflare.com URL or your own domain.
